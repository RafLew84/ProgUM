\chapter{Funkcje (Wykład 3)}

W Kotlinie funkcje to \textit{obywatele pierwszej kategorii} (\textit{first-class citizens}). Oznacza to, że funkcje nie są tylko nazwanymi blokami kodu wewnątrz klas (jak metody np. w Javie). Są to pełnoprawne obiekty, które możemy przekazywać jako parametry, zwracać z innych funkcji i przechowywać w zmiennych.

\section{Fundamenty Funkcji}

Definicja funkcji w Kotlinie jest prosta i czytelna. Używamy słowa kluczowego \texttt{fun}.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
fun double(x: Int): Int {
    return 2 * x
}
\end{minted}

Typ zwracany podajemy po dwukropku. Jeśli funkcja nic nie zwraca (czyli zwraca \texttt{Unit}), możemy ten typ pominąć.


Kotlin wprowadza \textbf{wartości domyślne} parametrów.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
fun connect(
    url: String, 
    timeout: Int = 5000, 
    retry: Boolean = true
) { /*...*/ }
\end{minted}

Teraz możemy wywołać tę funkcję na wiele sposobów, korzystając z mechanizmu \textbf{argumentów nazwanych}. Pozwala on na podawanie parametrów w dowolnej kolejności, o ile nazwiemy je tak, jak w definicji.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
connect("http://example.com") // timeout=5000, retry=true
connect("http://example.com", retry = false) // timeout=5000
connect(timeout = 1000, url = "http://example.com") // Kolejność nie ma znaczenia
\end{minted}

Jeśli potrzebujemy funkcji przyjmującej dowolną liczbę argumentów (np. lista ID do usunięcia), używamy modyfikatora \texttt{vararg}. Wewnątrz funkcji taki parametr jest widoczny jako tablica (\texttt{Array<T>}).

\begin{minted}[frame=single,framesep=10pt]{kotlin}
fun printScores(vararg scores: Int) {
    for (score in scores) {
        print("$score ")
    }
}

printScores(1, 2, 3, 4, 5)
\end{minted}

\section{Funkcje w stylu Kotlin (Idiomy)}

Kotlin oferuje mechanizmy, które pozwalają pisać kod czytany niemal jak proza. Dwa z nich zasługują na szczególną uwagę: funkcje rozszerzające (Extensions) oraz funkcje infiksowe (Infix).

Funkcje rozszerzające pozwalają \textbf{dodać nową funkcję do istniejącej klasy} bez konieczności dziedziczenia z niej ani używania wzorca Dekorator. Możemy rozszerzać nawet klasy, których nie posiadamy kodu (np. \texttt{String} z biblioteki standardowej JDK).

Definiujemy je, podając nazwę typu przed nazwą funkcji:
\begin{minted}[frame=single,framesep=10pt]{kotlin}
// Rozszerzamy klasę String o metodę isEmail()
fun String.isEmail(): Boolean {
    // 'this' odnosi się do instancji Stringa, na którym wywołujemy funkcję
    return this.contains("@") && this.contains(".")
}

val email = "user@example.com"
if (email.isEmail()) { 
    println("Poprawny email")
}
\end{minted}

Pod maską, kompilator zamienia to na statyczną funkcję, która przyjmuje obiekt jako pierwszy parametr. Dzięki temu rozszerzenia nie łamią hermetyzacji (nie mają dostępu do prywatnych pól klasy).

Funkcje infiksowe pozwalają wywoływać funkcje (które mają dokładnie jeden parametr) w sposób przypominający operatory matematyczne lub język naturalny.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
infix fun Int.times(str: String): String {
    return str.repeat(this)
}

// Wywołanie standardowe
3.times("Hello") 

// Wywołanie infiksowe - jak zdanie
3 times "Hello"
\end{minted}

Infixy są powszechnie stosowane w DSL-ach (Domain Specific Languages) oraz w testach (np. biblioteka AssertK: \texttt{assertThat(price) isEqualTo 100}).

\section{Funkcje Wyższego Rzędu i Lambdy}

Funkcja wyższego rzędu to taka funkcja, która przyjmuje inną funkcję jako parametr lub ją zwraca. To fundament programowania funkcyjnego.

Wyrażenia lambda to funkcja anonimowa (bez nazwy), którą możemy zapisać w zwięzły sposób.

Pełna składnia:
\begin{minted}[frame=single,framesep=10pt]{kotlin}
val sum: (Int, Int) -> Int = { x: Int, y: Int -> x + y }
\end{minted}

Uproszczona składnia (typ wnioskowany):
\begin{minted}[frame=single,framesep=10pt]{kotlin}
val sum = { x: Int, y: Int -> x + y }
\end{minted}

Kotlin posiada konwencję, która bardzo upraszcza kod:
1. Jeśli funkcja jest ostatnim parametrem wywołania, lambdę możemy wyciągnąć \textbf{poza nawias}.
2. Jeśli lambda ma tylko jeden parametr, nie musimy go nazywać – jest on dostępny pod domyślną nazwą \texttt{it}.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
val numbers = listOf(1, 2, 3, 4, 5)

// Wersja "długa"
numbers.filter({ x -> x > 2 })

// Wersja idiomatyczna (trailing lambda + it)
numbers.filter { it > 2 }
\end{minted}

To dlatego kod Compose (\texttt{Column \{ ... \}}) czy Gradle (\texttt{dependencies \{ ... \}}) tak wygląda. Te nawiasy klamrowe to właśnie lambdy.

\section{Referencje do Funkcji (Operator \texttt{::})}

Operator \texttt{::} służy do tworzenia referencji do funkcji lub właściwości. Pozwala to przekazywać istniejące funkcje jako argumenty do innych funkcji (np. funkcji wyższego rzędu) zamiast definiować nową lambdę.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
fun isEven(n: Int): Boolean = n % 2 == 0

val numbers = listOf(1, 2, 3, 4, 5, 6)

// Przekazanie referencji do funkcji zamiast lambdy { isEven(it) }
val evenNumbers = numbers.filter(::isEven)

// Referencja do metody klasy
val list = listOf("a", "bc", "def")
val lengths = list.map(String::length)
\end{minted}

\section{Funkcje Anonimowe}

Funkcje anonimowe to funkcje, które nie mają przypisanej nazwy. Są one wyrażeniem, podobnie jak lambdy, ale ich składnia jest bliższa tradycyjnym funkcjom.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
val sum = fun(x: Int, y: Int): Int {
    return x + y
}
\end{minted}

Kluczowe cechy i różnice względem lambd:
\begin{itemize}
    \item \textbf{Instrukcja \texttt{return}}: W funkcjach anonimowych słowo kluczowe \texttt{return} działa tak samo, jak w funkcjach nazwanych – zwraca wartość z samej funkcji anonimowej. W lambdach \texttt{return} bez etykiety próbuje zwrócić wartość z funkcji otaczającej (non-local return).
    \item \textbf{Jawność typów}: Funkcje anonimowe wymagają jawnej deklaracji parametrów oraz typu zwracanego (przy użyciu ciała blokowego).
    \item \textbf{Zastosowanie}: Są użyteczne, gdy potrzebujemy precyzyjnej kontroli nad typem zwracanym lub gdy chcemy uniknąć niejasności związanych z działaniem \texttt{return} wewnątrz lambdy.
\end{itemize}

\section{Funkcje Zakresu (Scope Functions)}

To zestaw 5 funkcji standardowych: \texttt{let}, \texttt{run}, \texttt{with}, \texttt{apply}, \texttt{also}. Wszystkie robią coś bardzo podobnego: wykonują blok kodu na obiekcie. Różnią się tylko tym, jak ten obiekt jest dostępny w środku (\texttt{this} czy \texttt{it}) i co zwracają (sam obiekt czy wynik lambdy).

Oto \textit{ściąga}, jak wybierać odpowiednią funkcję:

\begin{enumerate}
    \item \textbf{\texttt{apply}}: Konfiguracja obiektu. Dostęp przez \texttt{this}, zwraca \textbf{obiekt}.
    \begin{minted}[frame=single,framesep=10pt]{kotlin}
    val user = User().apply {
        name = "Jan"
        age = 25
        city = "Kraków"
    }
    \end{minted}
    
    \item \textbf{\texttt{let}}: Praca z obiektami nullowalnymi lub transformacja. Dostęp przez \texttt{it}, zwraca \textbf{wynik lambdy}.
    \begin{minted}[frame=single,framesep=10pt]{kotlin}
    val name: String? = "Kotlin"
    val length = name?.let { 
        println("Przetwarzanie: $it")
        it.length // wynik lambdy przypisany do length
    }
    \end{minted}
    
    \item \textbf{\texttt{also}}: Wykonywanie dodatkowych operacji, tzw. \textit{efektów ubocznych} (akcji, które nie zmieniają samego obiektu, ale wpływają na świat zewnętrzny, np. logowanie lub zapis do bazy), bez przerywania łańcucha wywołań. Dostęp przez \texttt{it}, zwraca \textbf{obiekt}.
    \begin{minted}[frame=single,framesep=10pt]{kotlin}
    val numbers = mutableListOf(1, 2, 3)
        .also { println("Log: utworzono listę $it") }
        .apply { add(4) }
    \end{minted}
    
    \item \textbf{\texttt{run}}: Połączenie \texttt{with} i \texttt{let}. Uruchomienie bloku kodu na obiekcie i zwrócenie wyniku. Dostęp przez \texttt{this}.
    \begin{minted}[frame=single,framesep=10pt]{kotlin}
    val result = "Hello World".run {
        println("Długość tekstu: $length")
        length * 2
    }
    \end{minted}
    
    \item \textbf{\texttt{with}}: Podobne do \texttt{run}, ale obiekt podajemy jako argument. Używane, gdy chcemy wykonać wiele operacji na jednym obiekcie bez powtarzania jego nazwy.
    \begin{minted}[frame=single,framesep=10pt]{kotlin}
    with(user) {
        println("Imię: $name")
        println("Wiek: $age")
        saveToDatabase()
    }
    \end{minted}
\end{enumerate}

\section{Zaawansowane Koncepcje}

\subsection{Funkcje Inline}
Każda lambda w Kotlinie jest kompilowana do anonimowej klasy. Oznacza to, że każde jej wywołanie wiąże się z utworzeniem nowego obiektu, co zajmuje pamięć i czas procesora. W przypadku funkcji wywoływanych miliony razy (np. w pętlach), narzut ten staje się zauważalny.

Słowo kluczowe \texttt{inline} to instrukcja dla kompilatora: \textit{Nie twórz obiektu dla tej lambdy. Zamiast tego, skopiuj kod tej funkcji i \textbf{wklej go} („kopiuj-wklej”) bezpośrednio w miejsce wywołania}.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
inline fun executeOp(op: () -> Unit) {
    op() // Treść 'op' zostanie tu wklejona
}
\end{minted}

Wyobraź sobie, że piszesz książkę (program).
\begin{itemize}
    \item Zwykła funkcja to przypis (referencja): \textit{Patrz strona 45}. Czytelnik (procesor) musi przerwać czytanie bieżącego wątku, skoczyć na stronę 45, przeczytać i wrócić.
    \item Funkcja \texttt{inline} to operacja \textbf{Znajdź i Zamień} w edytorze. Edytor bierze treść ze strony 45 i fizycznie wkleja ją w każde miejsce, gdzie było odesłanie. Książka robi się grubsza (więcej kodu bajtowego), ale czyta się ją płynnie, bez ciągłego skakania po stronach.
\end{itemize}

\paragraph{noinline}
 Czasami funkcja \texttt{inline} przyjmuje kilka lambd, ale tylko niektóre z nich chcemy \textit{wkleić}. Inne mogą być np. przekazywane dalej do innych funkcji (które nie są inline) lub zapisywane w polach klas. W takiej sytuacji, \textit{wklejenie jest niemożliwe} – musimy mieć fizyczny obiekt lambdy. Używamy wtedy modyfikatora \texttt{noinline}.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
inline fun manyOperations(
    op1: () -> Unit,
    noinline op2: () -> Unit // Ta lambda pozostanie obiektem
) {
    op1()
    someExternalStore(op2) // Nie można przekazać kodu "wklejonego", trzeba obiektu
}
\end{minted}

Załóżmy, że mamy dwa dokumenty. Jeden przepisujemy ręcznie do głównego zeszytu (\texttt{inline}), ale drugi musimy włożyć do koperty i wysłać pocztą do urzędu (\texttt{noinline}). Nie możesz wysłać \textit{przepisanej treści} – urząd wymaga oryginału (obiektu w kopercie).

\paragraph{crossinline}
Jak wspomnieliśmy, w funkcjach \texttt{inline} dozwolony jest \texttt{return} (powrót nielokalny). Czasami jednak lambda jest wykonywana w innym kontekście (np. wewnątrz zagnieżdżonej funkcji lub innego wątku), gdzie nagłe przerwanie funkcji zewnętrznej byłoby niebezpieczne lub niemożliwe.

Słowo kluczowe \texttt{crossinline} blokuje możliwość użycia \texttt{return} w lambdzie, mimo że funkcja jest \texttt{inline}. Pozwala to na bezpieczne użycie lambdy w zagnieżdżonych kontekstach.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
inline fun executeLater(crossinline body: () -> Unit) {
    val runnable = Runnable {
        body() // Wywołanie w innym kontekście – non-local return zabroniony
    }
    runnable.run()
}
\end{minted}

Załóżmy że jesteśmy w pokoju (funkcja).
\begin{itemize}
    \item Zwykły \texttt{return} to wyjście głównymi drzwiami.
    \item Lambda \texttt{crossinline} to wejście do mniejszej, zamkniętej kabiny wewnątrz tego pokoju. Obiecujemy kompilatorowi (\texttt{crossinline}), że nie spróbujemy \textit{wyskoczyć przez okno} (return) bezpośrednio z kabiny na zewnątrz budynku, pomijając drzwi pokoju. Możemy zakończyć tylko zadanie w kabinie (\texttt{return@label}).
\end{itemize}

\subsection{Rekursja ogonowa (tailrec)}
Rekursja grozi przepełnieniem stosu (\texttt{StackOverflowError}). Kotlin oferuje optymalizację: jeśli wywołanie rekurencyjne jest \textbf{ostatnią} operacją w funkcji, możemy dodać modyfikator \texttt{tailrec}. Kompilator zamieni wtedy rekurencję na zwykłą pętlę, co zapobiega przepełnieniu stosu.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
tailrec fun silnia(n: Int, run: Int = 1): Int {
    return if (n == 1) run else silnia(n - 1, n * run)
}
\end{minted}

\section{Funkcje Generyczne}

Funkcje generyczne pozwalają na tworzenie kodu \textit{szablonowego}, który może pracować z różnymi typami danych, zachowując pełne bezpieczeństwo typowania i unikając rzutowania. Typ generyczny definiujemy w nawiasach ostrych \texttt{<T>} przed nazwą funkcji.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
fun <T> singletonList(item: T): List<T> {
    return listOf(item)
}
\end{minted}

Czasami nasz szablon ma sens tylko dla pewnej grupy typów (np. tylko dla liczb). Możemy to wymusić po dwukropku:

\begin{minted}[frame=single,framesep=10pt]{kotlin}
// T musi dziedziczyć po Number (Int, Double, Float...)
fun <T : Number> doubleValue(value: T): Double {
    return value.toDouble() * 2
}
\end{minted}

Jeśli potrzebujemy kilku warunków jednocześnie (np. \textit{musi być liczbą} oraz \textit{dać się porównywać}), stosujemy słowo kluczowe \texttt{where}:

\begin{minted}[frame=single,framesep=10pt]{kotlin}
fun <T> sortNumbers(list: List<T>)
    where T : Number, T : Comparable<T> {
    // ...
}
\end{minted}

W Javie (i standardowo w Kotlinie) typy generyczne podlegają \textbf{wymazywaniu typów} (\textit{type erasure}). Oznacza to, że w trakcie działania programu \texttt{List<String>} i \texttt{List<Int>} są tym samym: po prostu \texttt{List}. Wirtualna maszyna nie wie, co było w środku. Dlatego nie możemy napisać: \texttt{if (obj is T)}.

Jednak dzięki funkcjom \texttt{inline}, możemy użyć specjalnego słowa kluczowego \texttt{reified}. Ponieważ kod funkcji jest \textit{wklejany} w miejsce wywołania, kompilator w tym konkretnym miejscu \textbf{zna} dokładny typ i może go \textit{utrwalić} (reify).

\begin{minted}[frame=single,framesep=10pt]{kotlin}
// Zwykła funkcja - to nie zadziała:
// fun <T> checkType(a: Any) = a is T // Błąd kompilacji! Erasure.

// Funkcja inline + reified
inline fun <reified T> isType(a: Any): Boolean {
    return a is T // Teraz działa
}

val isString = isType<String>("Tekst") // true
val isInt = isType<Int>("Tekst")       // false
\end{minted}

Posłużmy się analogią:
\begin{itemize}
    \item Zwykłe generyki to list w zaklejonej kopercie. Listonosz (runtime) wie, że niesie \textit{List}, ale nie wie co jest w środku (String czy Int), bo koperta jest nieprzezroczysta (\textit{erasure}).
    \item \texttt{reified} w funkcji \texttt{inline} to sytuacja, w której nadawca (kompilator) otwiera kopertę tuż przed wręczeniem jej tobie w miejscu odbioru. Widzisz zawartość, więc wiesz dokładnie, co to za typ.
\end{itemize}

\section{Przeciążanie Operatorów (operator fun)}

Kotlin umożliwia dostarczenie własnych implementacji dla ustalonego zestawu operatorów (takich jak \texttt{+}, \texttt{*}, \texttt{==}, czy \texttt{[]}). Aby funkcja mogła pełnić rolę operatora, musi być poprzedzona słowem kluczowym \texttt{operator} i posiadać konkretną, zarezerwowaną nazwę.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
data class Vector(val x: Int, val y: Int) {
    // Przeciążenie operatora +
    operator fun plus(other: Vector): Vector {
        return Vector(x + other.x, y + other.y)
    }
}

val v1 = Vector(1, 2)
val v2 = Vector(3, 4)
val v3 = v1 + v2 // Wynik: Vector(x=4, y=6)
\end{minted}

Najpopularniejsze operatory to:
\begin{itemize}
    \item \texttt{plus}, \texttt{minus}, \texttt{times}, \texttt{div}, \texttt{rem} (arytmetyczne).
    \item \texttt{get}, \texttt{set} (dostęp przez nawiasy kwadratowe \texttt{[]}).
    \item \texttt{invoke} (pozwala wywołać obiekt jak funkcję: \texttt{obiekt()}).
    \item \texttt{contains} (operator \texttt{in}).
\end{itemize}
