\chapter{Inicjalizacja i Delegacja (Wykład 7)}

W tym rozdziale zajmiemy się mechanizmami, które pozwalają na efektywne zarządzanie inicjalizacją właściwości oraz przekazywaniem odpowiedzialności za ich obsługę do innych obiektów. Poznamy słowa kluczowe \texttt{lateinit}, \texttt{by lazy} oraz wzorzec Delegata.

\section{Opóźniona Inicjalizacja (\texttt{lateinit})}

W Kotlinie właściwości klasy, które nie są nullowalne (nie mają \texttt{?}), muszą zostać zainicjalizowane w konstruktorze lub od razu przy deklaracji. Czasami jednak nie znamy wartości początkowej w momencie tworzenia obiektu – np. gdy jest ona wstrzykiwana przez framework (np. Dependency Injection w Androidzie) lub ustawiana w metodzie \texttt{setUp()} w testach jednostkowych.

W takich sytuacjach używamy modyfikatora \texttt{lateinit}.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
class MyService {
    lateinit var database: Database // Nie musimy od razu przypisywać wartości
    
    fun init() {
        database = Database() // Inicjalizacja później
    }
    
    fun query() {
        // Jeśli wywołamy to przed init(), dostaniemy wyjątek!
        database.query() 
    }
}
\end{minted}

\textbf{Ograniczenia:}
\begin{itemize}
    \item Tylko dla zmiennych \texttt{var}.
    \item Nie można używać z typami prostymi (prymitywnymi) jak \texttt{Int}, \texttt{Double}, \texttt{Boolean}.
    \item Typ nie może być nullowalny.
\end{itemize}

Aby upewnić się, czy zmienna została zainicjalizowana, możemy użyć referencji \texttt{.isInitialized}:
\begin{minted}[frame=single,framesep=10pt]{kotlin}
if (::database.isInitialized) {
    println("Baza gotowa")
}
\end{minted}

\section{Delegaty (Delegated Properties)}

Delegacja właściwości to mechanizm, w którym obsługa metod \texttt{get()} (i ew. \texttt{set()}) jest przekazywana do innego obiektu. Składnia wykorzystuje słowo kluczowe \texttt{by}.

\subsection{Inicjalizacja Leniwa (\texttt{by lazy})}

Najpopularniejszym delegatem jest \texttt{lazy}. Pozwala on na obliczenie wartości właściwości dopiero przy jej \textbf{pierwszym użyciu}. Wynik jest zapamiętywany i zwracany przy kolejnych odwołaniach. Jest to idealne dla kosztownych operacji (np. wczytywanie pliku, konfiguracja DB).

\begin{minted}[frame=single,framesep=10pt]{kotlin}
val heavyObject: Heavy by lazy {
    println("Obliczam...")
    Heavy() // To wykona się tylko raz
}

fun main() {
    println("Start")
    // Tu heavyObject jeszcze nie istnieje
    println(heavyObject) // "Obliczam..." -> zwraca obiekt
    println(heavyObject) // Zwraca zapamiętany obiekt (bez "Obliczam...")
}
\end{minted}

Działa tylko dla zmiennych \texttt{val}. Domyślnie jest bezpieczny wątkowo (\texttt{synchronized}).

\subsection{Obserwowanie zmian (\texttt{Delegates.observable})}

Możemy reagować na każdą zmianę wartości właściwości.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
import kotlin.properties.Delegates

var user: User by Delegates.observable(User("Pusty", 0)) { prop, old, new ->
    println("Zmiana właściwości ${prop.name}: $old -> $new")
}
\end{minted}
Każde przypisanie nowej wartości do \texttt{user} spowoduje wypisanie logu.

\subsection{Wetowanie zmian (\texttt{Delegates.vetoable})}

Pozwala zablokować zmianę wartości, jeśli nie spełnia ona określonego warunku.

\begin{minted}[frame=single,framesep=10pt]{kotlin}
var age: Int by Delegates.vetoable(0) { prop, old, new ->
    new >= 0 // Akceptujemy tylko wiek nieujemny
}

age = 10
println(age) // 10
age = -5     // Zmiana odrzucona!
println(age) // 10
\end{minted}

\subsection{Delegowanie do Mapy}

Ciekawym zastosowaniem jest delegowanie właściwości do mapy. Jest to przydatne przy parsowaniu dynamicznych struktur danych (np. JSON).

\begin{minted}[frame=single,framesep=10pt]{kotlin}
class User(val map: Map<String, Any?>) {
    val name: String by map
    val age: Int by map
}

val data = mapOf("name" to "Jan", "age" to 25)
val u = User(data)
println(u.name) // "Jan"
\end{minted}
Klucze w mapie muszą odpowiadać nazwom właściwości.

\section{Podsumowanie}

\begin{itemize}
    \item Używaj \texttt{lateinit} dla \texttt{var}, gdy nie możesz zainicjalizować pola w konstruktorze (np. wstrzykiwanie zależności), ale masz pewność, że zrobisz to przed użyciem.
    \item Używaj \texttt{by lazy} dla \texttt{val}, gdy obliczenie wartości jest kosztowne i chcesz je opóźnić do momentu pierwszego użycia.
    \item Delegaty \texttt{observable} i \texttt{vetoable} pozwalają na łatwą implementację logiki monitorowania i walidacji zmian danych.
\end{itemize}
